Introduction¶

On April 15, 1912, during her maiden voyage, the Titanic sank after colliding with an iceberg, killing 1502 out of 2224 passengers and crew. This sensational tragedy shocked the international community and led to better safety regulations for ships.

Feature Engineering¶

Feature engineering is the process of using domain knowledge of the data to create features that make machine learning algorithms work. Feature engineering is fundamental to the application of machine learning, and is both difficult, time-consuming and requires expert knowledge.

  • A feature is a piece of information that might be useful for prediction. Any attribute could be a feature, as long as it is useful to the model.

  • The quality and quantity of the features will have great influence on whether the model is good or not, so choosing the right features is very important.

  • Better features can produce simpler and more flexible models, and they often yield better results.

Joy of supervised machine learning is that it is an iterative process, we can create and test many new features and discard if not needed. In this particular case, we engineer nine new features.

  • Family Size - Number of family members aboard ship

  • Person Type - Child if 14 or under, male adult or female adult

  • Family Group - Surname

  • Ticket Type - Individual, family, friends or family/friends (based on ticket info)

  • Passengers on Ticket - Number of passengers travelling on ticket

  • Singleton - True if travelling on individual ticket and no family aboard

  • Deceased Child in Group - True if we know that a child in group (Surname or Ticket) died, false if not

  • Deceased Female in Group - True if we know that a female in group (Surname or Ticket) died, false if not

  • Surviving Male in Group - True if we know that a male in group (Surname or Ticket) survived, false if not

Random Forest Model¶

Random forests are an ensemble learning method for classification and regression that operate by constructing many decision trees during training. Each tree is constructed from a bootstrapped version of the original training data and a random subset of variables are selected for splitting at each node. These trees are then bagged together and the final prediction is a function of each individual tree's output (commonly the mode for classification).

By averaging multiple decision trees, trained on different parts of the same training set, the goal is to overcome the over-fitting (high variance) problem of an individual decision tree. In practice this comes at the expense of a small increase in the bias and some loss of interpretability, but generally greatly boosts the performance of the final model.

As well as being a strong out-of-the-box classification solution, random forests allow us to compute the relative importance of each attribute, useful for feature selection in future models.

Ensemble methods are learning algorithms that construct a set of classifiers and then classify new data points by taking a (weighted) vote of their predictions

A decision tree is a set of rules used to classify data into categories. It looks at the variables in a data set, determines which are most important, and then comes up with a tree of decisions which best partitions the data.

Results¶

We constructed a random forest classifier using 2000 trees, tuning the min_samples_leaf parameter with a cross validated grid search to find the optimal parameter settings.

  • Testing classification accuracy score of 0.82

Conclusions¶

  • High classification accuracy with a relatively simple model, currently 140th out of 6000+ teams in competition leaderboard. Further analysis could focus on different machine learning models including SVMs, neural networks etc.
  • Highlights importance of Feature Engineering. The variable _deceased_female_ingroup shown to be second most important variable. Further engineering could greatly improve model's predictive capability - for example capturing the social networks onboard the Titanic.
  • Our primary performance measure here was classification accuracy, however it would be useful to see a more detailed breakdown of performance, for example, is the model better at correctly predicting survival (true positives) or death (false negatives)
