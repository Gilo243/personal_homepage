Life and death aboard the Titanic¶

An exploration into machine learning using the Kaggle Titanic dataset¶

By Greg Giles¶

The sinking of the RMS Titanic is one of the most infamous shipwrecks in history. On April 15, 1912, during her maiden voyage, the Titanic sank after colliding with an iceberg, killing 1502 out of 2224 passengers and crew. This sensational tragedy shocked the international community and led to better safety regulations for ships.

In this challenge, we are asked to complete the analysis of what sorts of people were likely to survive. This particular presentation focuses on using the tools of machine learning to predict which passengers survived the tragedy.

Exploring the Data¶

There are two datasets we will need to address, one for training our model and a second for testing purposes. For the initial data preparation and data cleaning, the datasets will be merged.

Now we can begin the exploration! Let's start by taking a look at the variables we have available.

Two datasets, train and test. Our survival prediction model will learn from the training data.

Load both train and test dataframes in and merge. Ideally keep the test data set isolated from training but for this Titanic problem, I am going to initially merge the two datasets to speed up imputation of missing values, feature engineering etc

To highlight the variables available I am going to pick out a Japanese gentleman named Masabumi Hosono.

Hosono - only Japanese passenger aboard the Titanic and despite the overwhelming odds he survived....only to lose his job and be condemned as a coward on his return to Japan.

Scikit learn, the package we will be using to build our classification model, does not accept string values as input, so for use in the model, we will need to have integer representation of our categorical variables. With simple dichotomous variables e.g. gender, we can use binarization, but for nominal multiclass variables we do not want to imply an order, so a method like one-hot encoding would be useful.

This means that dropping records with missing values is out of the question, will instead use imputation to fill in missing values.

Cabin - Missing value imputation¶

First up cabin, with approximately 1000 missing values out of 1300 there are far too many to impute with any real conviction. Coupled with this, a high percentage of the values we do have are unique and generally speaking this isn't good for prediction.

Age - Missing value imputation¶

As we noted earlier, there are quite a few missing Age values in our data, over 260 infact. This means we need to be be a bit cleverer with our imputation. To do this, we will introduce a new variable Title consisting of salutations parsed from the Name variable; this will help in both imputation and be an interesting variable for survival prediction.

Given the small size of the dataset, we probably should not opt for deleting either entire observations (rows) or variables (columns) containing missing values.

We’re left with the option of either replacing missing values with a sensible values given the distribution of the data, e.g., the mean, median or mode.

We could go with prediction and use other variables - mice etc

def ticket_type(row):
    if row['Passenger_count'] == 1:
        return 'individual'
    elif (row['Passenger_count'] > 1)&(row['Unique_Surnames'] == 1):
        return 'family'
    elif (row['Passenger_count'] > 1)&(row['Unique_Surnames'] == row['Passenger_count']):
        return 'friends'
    else:
        return 'family/friends'

def cv_scorer(model, folds, X, y, scoring= 'accuracy'):
    kf = sklearn.model_selection.StratifiedKFold(folds, True)
    scores = sklearn.model_selection.cross_val_score(model, X, y, scoring= scoring, cv = kf)
    return np.mean(scores)
